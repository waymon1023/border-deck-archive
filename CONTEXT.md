## 1. 项目定位

这是一个纯展示型静态网站，核心形式是：

- 首页展示一组扑克牌背面。
- 用户点击某张牌后，卡牌翻转。
- 翻转后进入该牌对应的“生存游戏档案”详情页。
- 每张牌包含：游戏名称、花色类型、难度、人数、场地、规则、通关条件、残忍点、场景图、图片提示词等。

项目灵感来自“扑克牌生存游戏”类型作品，但要做成原创设定站点，不使用官方 Logo、截图、剧照、漫画图、角色图或直接复制原作规则。

推荐网站定位：

> 一个受扑克牌生存游戏题材启发的原创互动图鉴网站。

## 2. 版权与素材原则

为了降低侵权风险：

- 不直接使用《弥留之国的爱丽丝 / Alice in Borderland》的官方 Logo。
- 不使用剧照、漫画截图、海报、角色图。
- 不照搬原作已有游戏的完整规则文案。
- 可以保留“扑克牌 + 花色分类 + 生存游戏 + 翻牌展示”的灵感层。
- 所有新增游戏设定、场景图、UI、卡牌背面、文案尽量原创。
- 图片建议使用 AI 生成原创场景图，或者使用明确可商用授权素材。

可以在网站底部加入类似声明：

> 本站为非商业性质的原创展示项目，灵感来源于扑克牌生存游戏题材。所有新增游戏设定、场景图、文案均为原创内容。本站与任何影视、漫画、游戏版权方无官方关联。

## 3. 四种花色的设计方向

### ♠ 黑桃：体能

关键词：

- 体能
- 耐力
- 速度
- 高空
- 追逐
- 身体极限
- 环境压迫

黑桃的残忍不是心理欺骗，而是身体撑不住。越到高牌，越应该体现“知道怎么赢，但身体做不到”。

### ♥ 红桃：心理 / 感情 / 欺诈

关键词：

- 信任
- 朋友
- 感情
- 欺骗
- 背叛
- 后劲
- 通关后的精神折磨

红桃不是单纯狼人杀。不是“谁在骗我”，而是“我知道他在骗我，但我还是想相信他”。K 级红桃应该让玩家靠欺骗最信任自己的人活下去，并在通关后产生精神反噬。

### ♦ 方片：纯智斗

关键词：

- 数学
- 逻辑
- 信息差
- 证明链
- 博弈
- 少数人生还
- 冷酷理性

方片不要感情、不要团队煽情，就是聪明的人活。高难方片应该是 1～3 人生还，靠信息差、推理、反制和证明链淘汰其他人。

### ♣ 梅花：团队合作 / 团队对抗

关键词：

- 团队协作
- 分工
- 资源运营
- 团队对抗
- 战术逆转
- 翻盘代价

梅花不是规则强制牺牲，而是团队对抗中，如果正常赢不了，想靠惊天逆转赢过对方团队，可能需要有人付出代价。牺牲不是必选项，而是落后时的高风险翻盘路线。

## 4. K 牌统一调性

A 到 K 的整体递进：

- A-3：理解规则
- 4-6：执行规则
- 7-9：利用规则
- 10-Q：反抗规则 / 高级博弈
- K：这个花色的终极考验

K 牌不是简单更难，而是每个花色的核心能力被逼到极限。

一句话概括四张 K：

- ♠K：身体上的绝望。
- ♥K：感情上的后遗症。
- ♦K：智力上的淘汰赛。
- ♣K：团队胜利里的缺口。

# 5. 四张 K 的最终设定

---

## ♠K《极限垂直》

### 定位

黑桃 K 是纯体能极限。它不靠心理、投票或隐藏身份，而是让玩家知道怎么赢，但身体做不到。

### 基本信息

- 花色：♠K
- 类型：体能 / 耐力 / 攀爬 / 极限生存
- 人数：13人
- 场地：废弃通讯塔 + 高空维修通道
- 时长：90分钟
- 通关人数：理论上不限，实际极少
- 通关条件：登上最高层并触碰终点信标

### 场地

一座接近废弃的超高通讯塔，从下到上分成五段：

1. 楼梯井，连续爬升
2. 断裂平台，需要跳跃
3. 外部钢架，高空强风
4. 垂直维修梯，体力爆炸消耗
5. 塔顶天线，只能徒手攀爬

没有复杂谜题，没有隐藏身份，没有投票，就是爬上去。

### 核心规则

每个人身上有腕带，显示三个数值：

- 心率
- 肌肉负荷
- 平衡稳定度

一旦某个数值超过临界点，系统会强制给玩家增加负重。

恶性循环：

> 越累 → 越慢 → 越紧张 → 心率越高 → 负重越重 → 更累

### K 级残忍点

黑桃 K 的残忍是身体开始背叛你。

终点就在上面，但手抖、腿软、呼吸乱、握不住梯子。你不是不知道怎么通关，而是清楚地知道自己可能真的爬不上去了。

### 关键机制：救援惩罚

玩家可以帮助别人，比如拉别人一把、扶住别人、替别人挡风、帮别人取下卡住的装备。

但每次救援，自己的腕带都会增加负荷值。

> 每救一个人，自己离失败就更近一步。

### 最终阶段

最后一段塔顶天线每次只能上一个人。下面的人可以抓住稳定绳，帮上面的人减少晃动。但抓绳的人会持续扣体力值。

可能出现的画面：

> 有人在上面冲终点。有人在下面死死拉住绳子。拉绳的人明知道自己可能没力气再上去了。

### 核心味道

体能、高空、疲劳、恐惧、极限判断。

## ♥K《亲友证明》

### 定位

红桃 K 不是狼人杀。狼人杀是“谁在骗我”，红桃 K 是“我知道他在骗我，但我还是想相信他”。

它的重点是朋友之间的感情欺诈，以及通关后的精神反噬。

### 基本信息

- 花色：♥K
- 类型：心理 / 感情 / 朋友欺诈 / 信任崩塌
- 人数：8人
- 场地：封闭式白色会客厅
- 时长：70分钟
- 通关人数：不固定
- 通关条件：拿到“亲友证明”并进入出口

### 初始规则

8 名玩家进入场地后，系统根据过往表现，把他们两两配对。

配对标准不是随机，而是系统判断双方之间信任值最高。配到一起的通常是：

- 一路走来的朋友
- 刚建立信任的搭档
- 曾经互救过的人
- 彼此最不想怀疑的人

每一对面前都有一张卡，叫“亲友证明”。

只有持有这张证明的人，才能进入出口。

系统告诉双方：

> 请在规定时间内决定，谁更应该活下去。

### 私人提示

每个玩家都会收到一条只有自己能看到的提示：

> 你无法通过主动索要证明通关。你必须让对方自愿把证明交给你。

抢不行，逼不行，只有骗到对方自愿给你才行。

于是游戏变成：

> 你要让最信任你的人，亲手把生路交给你。

### 中段机制：记忆揭露

每隔十分钟，房间会播放一段双方过往片段。

这些片段会被系统恶意剪辑，比如：

- 你曾经犹豫过要不要救他
- 你曾经怀疑过他
- 你曾经隐瞒过信息
- 你曾经为了自己做过一次不光彩的选择

这些片段不一定是假的，但会被剪得非常伤人。

系统不是让双方互猜身份，而是在一点点破坏信任基础。

### K 级残忍点

玩家慢慢会发现，直接说真话很难拿到证明。

最有效的通关方式反而是：

- 伪装成更无私的人
- 伪装成更愧疚的人
- 伪装成更值得被托付的人

你要演出一种：

> 我不想活，但你一定要让我活。

### 最终反噬

游戏结束后，通关者会收到完整录像。

录像显示：

> 对方最后其实已经看穿了你的谎言。

但他还是把证明交给了你。

原因可能是：

- 他不想戳穿你
- 他想让你活下去
- 他已经不在乎自己了
- 他想让你一辈子记住这件事

真正的后劲不是“我被骗了”，而是：

> 我骗了他。他知道我骗了他。但他还是选择救我。

### 核心味道

你越会利用感情，越容易活。但活下来以后，才知道对方也许比你更清醒。

## ♦K《三席》

### 定位

方片 K 是纯智斗。不要感情，不要团队感动，不要牺牲煽情。

核心是：

> 聪明的人活。不够聪明的人出局。

生还人数很少，最多 3 人，实际可能只有 1 人。

### 基本信息

- 花色：♦K
- 类型：逻辑 / 数学 / 信息博弈 / 推理
- 人数：13人
- 场地：圆形演算室
- 时长：80分钟
- 通关人数：最多3人
- 通关条件：坐上三张“王席”之一

### 场地

房间中央有三张椅子：

- 第一王席
- 第二王席
- 第三王席

周围有 13 个独立计算台。

大屏幕显示：

> 本场游戏最多允许三人通关。请证明你有资格坐上王席。

### 核心规则

游戏分为五轮。

每一轮系统会给所有人一道大型逻辑题，但每个人拿到的信息不同，比如：

- 有人拿到数字范围
- 有人拿到排除条件
- 有人拿到部分公式
- 有人拿到别人答案的真假提示
- 有人拿到陷阱规则

每轮结束后，玩家必须提交答案。

结果分为：

- 完全正确：获得王分
- 部分正确：保留资格
- 错误：失去一条命题权限

三次错误，直接出局。

### 方片 K 的关键

这个游戏不是大家一起把题做出来，因为最终只有三个席位。

信息交换很危险。你告诉别人一条关键信息，可能不是帮大家通关，而是在帮别人抢你的位置。

> 合作能更快接近答案。但合作对象可能拿着你的信息坐上王席。

### 中段机制：答案污染

从第三轮开始，玩家可以提交“伪证明”。

伪证明不是直接撒谎，而是逻辑上看起来成立，但缺少一个关键前提。

如果别人引用了你的伪证明，他的最终答案会被污染。

后半段房间会进入纯智斗状态：

- 真信息
- 假信息
- 半真信息
- 诱导信息
- 反诱导信息

没人打架，没人煽情，但每个人都在用脑子杀人。

### 最终规则

第五轮结束后，所有还没出局的人可以申请王席。

坐上王席必须提交三样东西：

1. 最终答案
2. 完整证明链
3. 指出至少一名玩家证明链中的漏洞

最后不是单纯做对题，而是证明：

> 我对，而且别人不如我对。

### 最狠的地方

最多三个人通关，但不保证一定有三个人。

如果只有一个人的证明链完整，那就只有一个人活。

如果所有人的证明都有漏洞，那就全员失败。

### 核心味道

没有感情分，没有团队分，没有道德分。只有逻辑、信息差、推理、反制。

> 你不是被人背叛死的。你是因为不够聪明，被证明没有资格活。

## ♣K《逆转方舟》

### 定位

梅花 K 是团队合作 + 团队对抗。

它不是规则强制牺牲，而是在团队对抗中，如果正常赢不了，想惊天逆转赢过对方团队，可能要有人付出代价。

牺牲不是必选项，而是落后时的翻盘路线。

### 基本信息

- 花色：♣K
- 类型：团队合作 / 团队对抗 / 资源运营 / 战术逆转
- 人数：20人
- 队伍：两队，每队10人
- 场地：双核心地下设施
- 时长：90分钟
- 通关条件：本队方舟启动进度达到100%
- 通关人数：胜队全员理论可通关

### 场地

场地分成左右两个阵营。

每队都有一套完整设施：

- 动力室
- 控制室
- 维修区
- 运输区
- 核心舱

两队目标一样：

> 比对方更快启动自己的方舟。

启动方舟需要完成多个协作任务：

- 搬运能源核心
- 修复管线
- 解锁控制台
- 运输零件
- 分配人手防守
- 干扰对方进度

这是纯梅花：一个人再强也没用，必须靠团队配合。

### 表面规则

胜队全员通关，败队全员失败。

大家一开始会以为：

> 只要团队更强，全员就能活。

### 核心机制：分工效率

每个玩家会被系统分配一个隐藏职能：

- 搬运型
- 计算型
- 维修型
- 指挥型
- 干扰型
- 支援型

如果队伍分工合理，进度很快。

如果所有人都抢着当指挥，或者没人愿意做苦活，进度会崩。

前半段是团队运营。

### 中段机制：对方干扰

两队可以派人进入公共区域争夺资源。

公共资源包括：

- 备用能源
- 维修组件
- 冷却液
- 权限卡
- 干扰装置

团队要决定：

- 几个人留守？
- 几个人进攻？
- 几个人偷资源？
- 几个人保护核心？

### 关键翻转：超频核心

最后 20 分钟，系统开放隐藏机制：

> 超频核心

任何一队都可以启动超频核心。

启动后，本队方舟进度瞬间提升 30%，并且所有任务效率翻倍。

如果本队落后很多，正常打已经赢不了，启动超频核心可能直接逆转。

### 代价

超频核心需要一个人进入核心舱内部手动维持。

维持期间，他不能离开。

如果他松手，超频失败。

如果超频完成，队伍大概率反超。

但核心舱在方舟启动后会被隔离。

> 规则没有要求你牺牲。但想靠超频翻盘，就得有人留下。

### 为什么不是强制牺牲

如果一个队前期配合特别好，完全可以不启动超频，正常赢，全员通关。

牺牲只会出现在这种情况：

> 前期落后。正常打赢不了。但还有一次逆转机会。这个机会需要一个队友留下。

残忍不是系统写死的，而是团队自己算出来的：

> 我们想全队赢。但我们赢的代价，是少一个人。

### 最终压力

最后几分钟，队伍内会产生争论：

- 要不要开超频？
- 谁进去？
- 是不是还有别的办法？
- 如果不开，我们是不是全输？
- 如果开了，那个人怎么办？

这不是红桃的个人欺骗，而是梅花的团队决策。

### 最好的戏剧点

留下的人不一定是最弱的。反而可能是团队里最关键的人：

- 指挥者
- 维修核心
- 最稳定的队友
- 前期带大家撑过来的人

因为只有他能维持超频。

团队会面对一个问题：

> 我们要不要为了赢，失去最像队长的那个人？

# 7. 技术落地方向

## 静态站即可

这个项目目前是纯展示型，适合静态部署：

- React / Vue / Next.js 静态导出
- 或纯 HTML / CSS / JS
- 部署到 Cloudflare Pages / Vercel / Netlify / GitHub Pages

## 推荐数据结构

建议把卡牌内容放在 JSON 或 TypeScript 数据文件里，例如：

```ts
export const cards = [
  {
    id: "spade-k",
    suit: "spade",
    rank: "K",
    title: "极限垂直",
    type: "体能 / 耐力 / 攀爬 / 极限生存",
    players: "13人",
    duration: "90分钟",
    location: "废弃通讯塔 + 高空维修通道",
    clearCondition: "登上最高层并触碰终点信标",
    coverImage: "/images/spade-k-cover.jpg",
    sections: [
      {
        title: "核心规则",
        content: "每个人身上有腕带，显示心率、肌肉负荷、平衡稳定度..."
      }
    ]
  }
]
```

## 推荐页面结构

- `/` 首页：扑克牌背面墙，点击翻牌
- `/cards/:id` 卡牌详情页
- `/about` 项目说明 / 版权声明
- `/gallery` 场地图合集，可选

## 推荐组件

- `CardGrid`：首页卡牌网格
- `PlayingCard`：单张翻牌组件
- `CardDetailPage`：详情页
- `GameInfoPanel`：基础信息面板
- `RuleSection`：规则模块
- `ImagePromptPanel`：图片提示词模块
- `DisclaimerFooter`：版权/非官方声明
